# КОМА — Комплексный Оптимизатор Методических Актов

**КОМА** (ранее *ТимПлан*) — это автоматизированная система на базе AI-агентов, предназначенная для извлечения данных из Рабочих Программ Дисциплин (РПД) в формате PDF и автоматического формирования Оценочных Материалов Дисциплин (ОМД) в формате Word (`.docx`).

Система гарантирует 100% сохранение стилистики, шрифтов и структуры исходного вузовского шаблона ОМД, автоматически генерируя академически точные вопросы и критерии оценивания на основе целевых компетенций.

---

## Ключевые возможности

1. **AI-Парсинг и Конвертация**: Использование парсера `docling` (IBM) для высокоточного распознавания сложных таблиц компетенций, часов и тем занятий из PDF-файла РПД в разметку Markdown.
2. **Шаг-за-шагом Конвейер**: Разделенная CLI/API архитектура позволяет запускать и контролировать каждый этап обработки независимо (с возможностью вернуться назад и скорректировать данные):
   - **Шаг 1: `parse`** — парсинг PDF в Markdown.
   - **Шаг 2: `extract_structure`** — извлечение метаданных, компетенций и сетки занятий в файл `variables.yml`.
   - **Шаг 3: `generate_questions`** — генерация академических вопросов и критериев для каждого занятия.
   - **Шаг 4: `generate_docx`** — компиляция данных в итоговый Word-документ.
3. **Двойная проверка качества (AI Quality Loops)**: Интегрированная связка агентов-исполнителей и критиков (`StructuralCritic`, `QuestionsCritic`) на базе Google Gemini (`gemini-2.5-flash`), которая проверяет соответствие результатов параметрам и структуре исходного документа для предотвращения галлюцинаций.
4. **Нормализация стилей**: Программное приведение финального документа Word к строгим стандартам ГОСТ (поля, выравнивание, размер шрифтов Times New Roman, форматирование заголовков и таблиц).

## Структура проекта

* `main.py` — основной интерфейс командной строки (CLI) для запуска шагов.
* `agents/` — AI-агенты на базе Google ADK (Agent Development Kit).
* `tools/` — низкоуровневые инструменты парсинга и сборки шаблонов docx.
* `schemas/` — Pydantic-схемы для строго структурированного вывода AI.
* `templates/` — исходный Word-шаблон ОМД для заполнения.
* `web/` — веб-интерфейс на базе фреймворка NiceGUI.
* `input/` — директория для исходных PDF-файлов (по проектам).
* `processing/` — промежуточные результаты обработки (`variables.yml`, логи, параметры).
* `results/` — готовые сгенерированные файлы Word.

## Быстрый запуск в Docker (Веб-интерфейс)

1. Создайте файл параметров `.env` в корневой директории:
   ```bash
   cp web/.env.example .env
   ```
2. Откройте `.env` и впишите ваш API-ключ Gemini в переменную `GOOGLE_API_KEY`.
3. Запустите контейнеры:
   ```bash
   docker compose up --build -d
   ```
4. Откройте веб-интерфейс в браузере: [http://localhost:8080](http://localhost:8080).
   * *Данные для входа по умолчанию:* логин `admin`, пароль `admin`.

## Использование CLI (Командной строки)

Вы можете запускать обработку локально на хосте или через контейнер:

```bash
python3 main.py --project <имя_проекта> --step <шаг> [--regenerate]
```

### Доступные шаги:
* `parse` — парсинг PDF в Markdown.
* `extract_structure` — сбор метаданных и таблиц в `variables.yml`.
* `generate_questions` — генерация вопросов на основе извлеченной структуры.
* `generate_docx` — сборка и оформление финального Word-файла.
* `all` — последовательный запуск всех шагов от начала до конца.

Использование флага `--regenerate` принудительно перезапустит обработку шага, очистив предыдущие промежуточные результаты.
